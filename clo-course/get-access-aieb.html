FAQ
Connecting on a different device or browser than the purchase? Here's where your key is
Same-browser connections need no key at all — the page connects by itself. On a different device or browser, the secure page asks for your license key. Find it in the receipt Lemon Squeezy emailed you right after you bought (from noreply@lemonsqueezy.com — check spam, or search your inbox for "Chief Leverage Officer" or "license"), or go to app.lemonsqueezy.com/my-orders, enter the email you bought with, click the magic link, and open your order — the key (with a copy button) is right there. Got your key in an email from Rashid (e.g. a free-access invite)? Use that one. Enter the key on the secure page only — never in Claude or any chat.
Do I need to know how to code?
No. You copy a few short commands into a chat box and click "Install." If you can copy and paste, you can do this — and the "Let Claude install it" prompt up top will walk you through it.
Why install a thin plugin plus MCP?
The thin plugin gives Claude Code setup guidance and a local proxy. The paid instructions stay on the hosted MCP server, so your subscription can be checked at runtime and updates can ship without reinstalling the whole skill library.
Does my license key work on more than one computer?
Your subscription allows a limited number of devices according to the Lemon Squeezy license settings. Each computer connects separately and stores its own opaque device token; the license key is not stored by the plugin.
Why does it need Node.js?
The local MCP proxy is a small Node.js script. It starts the secure connection, stores the device token locally, and forwards requests to the hosted MCP server.
How do updates work?
Most AIEB skill updates happen on the MCP server. You get the latest instructions automatically the next time a skill runs — no reinstall, nothing to do. You only reinstall the plugin when the local shell itself changes (new setup behavior, proxy, or hooks), and the plugin will tell you when that's needed: it checks the server for the current shell version at startup and, if yours is behind, prompts you with the exact update commands.
Can I use this in Claude Cowork?
Yes — no Claude Code needed. Add the AI Employee Builder plugin (Customize → Personal plugins → + → Add marketplace → the2hourclo/aieb-thin-plugin), then say set up my AI Employee Builder. Claude gives you a secure connection link — in the browser you bought in it connects by itself, and only on a different device does the page ask for your key (enter it there, not in chat). The connection finishes by itself a few seconds after the page says connected. If the connector doesn't light up, type /check-setup. Don't paste the MCP URL into Custom Connectors.
What if I cancel or my subscription lapses?
Cancelling revokes paid MCP access immediately. Renew or resume at Lemon Squeezy → My Orders, then run /setup-aieb to reconnect.
I already use Claude Code — can I skip ahead?
Yes. If the extension and Node.js are installed, tick Step 1 and jump to Step 2.
Is the Terminal method different from VS Code?
Same result, same commands — just a different place to type them. The VS Code way is click-based and friendly; Terminal is for command-line folks. Use the toggle at the top to switch.
