Start here — set up your workspace

Brand new? Two quick things — give your business a home folder, then run onboarding once. It works exactly the same in Claude Code and Claude Cowork.

First, make a folder for your business. Everything your AI Employees build lives inside one folder on your computer:

  1. Create a new folder anywhere you like — your Desktop is perfectly fine. Name it something like my-ai-business.
  2. Open it in your app. In Claude Code, open that folder in VS Code (File → Open Folder). In Cowork, start a session and point it at the folder.
  3. Then type the command below and answer its questions.

onboard me

Works in either app — just say: "onboard me"

What it does — three guided steps. You only answer questions; it does the building:

  1. Scaffolds your workspace — creates the authoring folders where every AI Employee you build will live.
  2. Maps your business — lays out your departments and a business map, so every AI Employee knows where to read from and write to.
  3. Builds your first skill — runs the Business X-Ray to find your highest-leverage job, then builds and tests your first AI Employee with you.

What it builds on disk — you don't create any of this by hand:

your-workspace/
├─ .claude/
│  ├─ skills/      ← the AI Employees you build
│  ├─ agents/      ← multi-step workers
│  ├─ commands/    ← shortcuts you can trigger
│  └─ hooks/       ← automatic checks that keep output on-brand
├─ .claude-state/  ← remembers your onboarding progress
├─ diagrams/       ← your Business X-Ray map (.drawio)
├─ marketing/  sales/  product/      ← your business, as folders
├─ finance/  operations/  strategy/    (your departments)
├─ BUSINESS-MAP.md ← the map your AI Employees read first
└─ CLAUDE.md       ← your workspace guide + skill routing
✅
That's the whole setup. When it finishes you'll have your workspace mapped and your first AI Employee running — that's Phases 1 and 2 below already done. Pick the path up from Phase 3.

3✓ Make it reliable — add a harnessYour skill works but sometimes drifts? Lock it in. You are here ▾

A new skill usually works — until one day the output wanders, or it ignores one of its own rules. The fix isn't to nag it with more instructions. It's a harness: an automatic check that runs every time and catches the slip before it reaches you. You don't write it — you tell Claude what "good" looks like and it builds the check.

When you're ready

Your skill works, but its output sometimes drifts, or it keeps breaking a rule you gave it (a banned word, a format, a length).

Do this

  • Run your skill a few times and note exactly where it slips (the rule it breaks).
  • Tell Claude to harden it — name the slip so the check knows what to catch.
  • Run it again and confirm the check catches the slip before you do.

Prompts to copy

Harden my [skill name] so it can't drift. Add an automatic check that runs every time.

My [skill name] keeps [the mistake — e.g. using em-dashes / going off-brand]. Add a harness that catches that before it reaches me.

Keep it reliable as it changes — add evals (tests that run every time, so it can't quietly regress later):

Generate evals for my [skill name] — turn a few real examples into tests and run them every time, so it can't silently regress.

Not triggering right? If it won't fire when it should, or fires for the wrong requests:

My [skill name] isn't triggering reliably. Optimize its description so it fires for the right requests and not the wrong ones.

Or just say: "harden this skill", "generate evals for this", or "fix this skill's triggering"

4✓ Connect skills into a systemOne skill needs another? Chain them into one flow. You are here ▾

A single job is often several steps: intake → do the work → check it → hand it off. When one skill needs another to run first, you chain them into a skill system — an AI Employee that does the whole job, not one slice of it.

When you're ready

You catch yourself running one skill, then copying its output into another. Those steps belong together.

Do this

  • Write down the steps of the bigger job, in the order they happen.
  • Tell Claude to build a skill system that runs those steps as one flow.
  • Run the whole chain end to end and watch it hand off cleanly between steps.

Prompts to copy

Build me a skill system for [the whole job]. The steps are: [step 1] then [step 2] then [step 3].

Chain my [skill A] and [skill B] into one flow so I don't have to move the work between them.

Or just say: "build a skill system for this"

5✓ Hand work to an agentWant it to run on its own and report back? You are here ▾

Some jobs you don't want to walk through step by step — you want to say "go do this and report back." That's an agent: it runs on its own, in the background or alongside other work, and hands you the result. Use it when supervising each step is the slow part.

When you're ready

The work could run in the background or in parallel, and you'd rather get a finished report than babysit it.

Do this

  • Pick a job you'd rather delegate than watch (research, a first pass, a batch).
  • Tell Claude to create an agent for it — say what "done" looks like.
  • Let it run, then read its report and adjust if needed.

Prompts to copy

Create an agent that [does the job] on its own and reports back when it's done. Here's what "done" looks like: [your standard].

Make a background agent for [the task] so I can keep working while it runs.

Or just say: "create an agent for this"

6✓ Make a shortcut — a commandDoing the same thing a lot? Give it one word. You are here ▾

When you run the same thing the same way over and over, stop re-explaining it. Turn it into a command — a /shortcut you run whenever you want it. You're still the one who kicks it off; it just saves you re-typing the whole request every time.

When you're ready

You've typed nearly the same request more than a few times. It wants a name.

Do this

  • Notice a request you make the same way again and again.
  • Tell Claude to make a command for it and pick a short name.
  • Run it by typing /yourcommand.

Prompts to copy

Make a command for [the routine] so I can run it by typing one short slash command.

Or just say: "make a command for this"

7✓ Put it on autopilot — hooks, loops, schedulesTake yourself out of the loop entirely. You are here ▾

A command still waits for you to type it — the last step removes even that. A hook fires automatically on an event (run a check after every save). A schedule runs the work on its own (every Monday at 9am). A loop repeats until something's done. Now the outcome happens without you starting it — whether or not you remember.

When you're ready

Something should happen on an event, or on a clock, without you starting it.

Do this

  • Decide what should run without you — on an event, or on a schedule.
  • Tell Claude to set it up (a hook for events, a schedule for a clock).
  • Confirm it runs on its own at least once.

Prompts to copy

Create a hook that [does the thing] automatically every time [the event happens].

Graduate my [skill name] to a scheduled routine — it's proven, so run it unattended [e.g. every Monday at 9am], add the safety guard rails, and send me the result.

Or just say: "create a hook for this" or "graduate this to a schedule"

Keep your employees sharp

As you add more AI Employees, give the whole set a tune-up now and then — catch overlap, fix drift, and tidy anything that's grown messy. This is part of running the loop, not a one-off.

Audit my skill library — find overlap, skill-soup, and anything that's grown messy, and tell me what to fix first.

Upgrade my skills based on that audit — clean up the structure and routing, and check each change with me before applying it.

Or just say: "audit my skills", then "upgrade my skills"

"When do I…" — quick cheat sheet

A fast gut-check for which tool fits. (Claude makes these calls with you — this is just so the words make sense.)

…split a skill?
When it's trying to do two jobs and keeps forgetting one. Two jobs → two skills that hand off.
…add a harness?
When a skill works but drifts, or breaks its own rule. An automatic check beats more instructions.
…use an agent?
When the work should run in the background or in parallel — "go do this and report back."
…make a command?
When you do the same thing the same way, often, and want a one-word shortcut.
…put it on autopilot?
When it should run on a schedule or fire on an event, with no one starting it.
…build the next one?
When the current employee is solid on all of the above. Map the next bottleneck and repeat.
